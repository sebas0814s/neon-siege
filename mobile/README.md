# Neon Siege — Mobile Android Build

> Branch: `mobile/android` · Target: Android APK via Capacitor

---

## Overview

This branch adapts Neon Siege for **Android phones** using [Capacitor](https://capacitorjs.com/) to wrap the game as a native APK. The game logic is the same as the web version; what changes is:

- **Touch controls** — virtual joystick (left thumb) + tap-to-shoot + action buttons
- **Landscape lock** — forced landscape orientation via `screen.orientation.lock()`
- **Optimized render** — reduced pixel ratio and fog distance for mid-range devices
- **Capacitor wrapper** — converts the HTML/JS game into a signed APK

---

## Architecture

```
neon-siege/
├── index.html              ← Game entry point (modified for mobile)
├── js/game.js              ← Game logic (detects touch, adapts input)
├── mobile/

├── capacitor.config.json   ← Capacitor app config

└── android/                ← Generated by `npx cap add android` (gitignored)
```

## How to Build the APK

### Prerequisites

| Tool | Version | Notes |
|------|---------|-------|
| Node.js | ≥ 18 | Required for Capacitor CLI |
| Java JDK | 17 | Required by Android Gradle |
| Android Studio | Latest | Required for SDK + build tools |
| Android SDK | API 34 | Set `ANDROID_HOME` env var |

### Step-by-step

```bash
# 1. Install Capacitor dependencies
npm install

# 2. Add the Android platform (first time only)
npx cap add android

# 3. Sync web assets into the Android project
npx cap sync android

# 4. Open in Android Studio to build / sign the APK
npx cap open android
```

In Android Studio:
- `Build > Build Bundle(s) / APK(s) > Build APK(s)` for a debug APK
- `Build > Generate Signed Bundle / APK` for a release APK

## Mobile-Specific Optimizations

- **Pixel ratio cap** — `renderer.setPixelRatio(Math.min(window.devicePixelRatio, 2))` prevents rendering at 3× on high-DPI screens
- **Reduced fog** — fog end reduced from 120 to 80 units on mobile
- **Touch event prevention** — `touchmove` preventDefault avoids scroll interference
- **Fullscreen on tap** — requests fullscreen API on game start

## Notes on `index.html` (mobile version)

The mobile `index.html` differs from the web version in two ways:

1. It loads `mobile/touch-controls.js` before `js/game.js`
2. The game loop reads `window.touchInput` for movement and look input when no mouse/keyboard is present

The touch integration is **non-breaking** — on desktop, `touch-controls.js` simply creates invisible DOM elements that never receive events.
